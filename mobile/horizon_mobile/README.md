# horizon_mobile

Flutter client for browsing Horizon digests from the published static feed.

## Data Source

By default the app loads:

`https://cdn.jsdelivr.net/gh/afetmin/Horizon@gh-pages/api/mobile-manifest.json`

That manifest is generated by Horizon and published to the `gh-pages` branch with the digest markdown files. The app then fetches each digest body from the `item.url` field in the manifest.

Default CDN behavior:

- Primary: `cdn.jsdelivr.net`
- Fallback: `cdn.jsdmirror.com`
- Custom manifest URLs entered in the app are used as-is and do not get automatic CDN rewriting

## Running Locally

```bash
flutter run --dart-define=MANIFEST_URL=https://cdn.jsdelivr.net/gh/<owner>/<repo>@gh-pages/api/mobile-manifest.json
```

## Notes

- `docs/api/mobile-manifest.json` is the mobile entrypoint generated by the Python pipeline.
- `docs/_posts/YYYY-MM-DD-summary-{lang}.md` remains the detail content format consumed by the app.
